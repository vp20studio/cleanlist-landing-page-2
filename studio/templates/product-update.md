# Product Update Template

Use this template for announcing new features, improvements, and releases.

## Frontmatter

```yaml
---
title: "[Feature Name]: [Brief Benefit Statement]"
description: "[150-160 char description with key benefit and what's new]"
date: "YYYY-MM-DD"
category: "product-updates"
tags: ["feature", "update", "relevant-product"]
author:
  name: "Cleanlist Team"
  role: "Product Team"
featuredImage:
  src: "/images/blog/feature-screenshot.png"
  alt: "Screenshot of new feature"
keywords: ["primary keyword", "secondary keyword"]
---
```

## Structure

### 1. Opening Hook (50-100 words)
- Lead with the problem this solves
- Mention the feature name
- State the core benefit

Example:
> "Managing enrichment costs shouldn't feel like guesswork. Today, we're launching Usage Dashboardâ€”a real-time view into your credits, API calls, and ROI across every data source."

### 2. What's New (200-300 words)
- Bullet points of key features
- Screenshot or GIF of feature in action
- Brief explanation of each capability

### 3. How It Works (200-300 words)
- Step-by-step walkthrough
- Use numbered list
- Include screenshots at key steps

### 4. Why It Matters (100-150 words)
- Connect back to user pain points
- Include a metric or stat if possible
- Quote from beta user if available

### 5. Getting Started (50-100 words)
- Where to find the feature
- Link to docs if applicable
- CTA to try it

### 6. What's Next (50 words)
- Tease upcoming features
- Invite feedback

---

## Word Count Target: 800-1,200 words

## Internal Links to Include:
- Link to relevant product page
- Link to pricing (if feature impacts plans)
- Link to related blog post

## SEO Checklist:
- [ ] Feature name in title
- [ ] Benefit in meta description
- [ ] "New feature" or "announcement" in H2
- [ ] Screenshot with descriptive alt text
